# Coursework2-Job-search
Курсовая по 4 модулю. Поиск вакансий с использованием API-сервиса hh.ru

## Описание
В этом проекте вам предстоит написать интеграцию с внешним сервисом.
Напишите программу, которая будет получать информацию о вакансиях с платформы hh.ru в России, сохранять ее в файл и позволять удобно работать с ней: добавлять, фильтровать, удалять.

## Информация для пользователей
### Документация для сбора вакансий с hh.ru
Ссылка на API: https://github.com/hhru/api/.


### В настоящее время реализован следующий функционал:
+ В модуле **apies.py** представлены классы для взаимодействия с API-сервисами:
    * создан абстрактный класс `AbstractAPI` с абстрактным методом `get_vacancies()` и `check_private_method()`, который проверяет наличие приватного метода в дочерних классах;
    * представлен дочерний класс `HeadHunterAPI` для работы с API-сервисом сайта _hh.ru_.
    Содержит приватный метод __connect_to_api переопределенный метод `get_vacancies()`, возвращающий JSON-ответ со списком вакансий.
+ В модуле **vacancy.py** представлен класс Vacancy для работы с вакансиями, полученными по запросу пользователя:
    * в классе определены атрибуты, такие как название вакансии (`title`), ссылка на вакансию (`url`), зарплата (`salary`) и краткое описание или требования (`description`);
    * реализованы методы валидации данных для приватных атрибутов и вывод диапазона зарплат вакансий;
    * представлены методы расчета средней зарплаты для сравнения оных у разных вакансий;
    * реализован класс-метод `cast_to_object_list()` для преобразования данных API в список объектов Vacancy;
    * есть строковое представление вакансии для показа пользователю.
+ В модуле **reports.py** представлены классы для сохранения информации о вакансиях и работы с файлами:
    * определён абстрактный класс `AbstractSaver` для работы с входящими данными. 
    Содержит абстрактные методы `add_vacancy()`, `get_vacancies()` и `delete_vacancy()` для взаимодействия с экземплярами вакансий.
    * создан дочерний класс `JSONSaver` для хранения информации в JSON-файле. В нём переопределены методы абстрактного класса, а также добавлены:
        - приватный атрибут названия файла (`filename`), который по умолчанию сохраняет данные в директорию `/data/vacancies.json`;
        - метод `load_data()` для загрузки данных из файла,
        - метод `save_data()` для сохранения данных в файл,
        - метод `clear_file()` для удаления всего JSON-файла.
    * создан дочерний класс `CSVSaver` для хранения информации в CSV-файле. В нём переопределены методы абстрактного класса, а также добавлены:
        - приватный атрибут названия файла (`filename`), который по умолчанию сохраняет данные в директорию `/data/vacancies.csv`;
        - метод `load_data()` для загрузки данных из файла,
        - метод `save_data()` для сохранения данных в файл,
        - метод `clear_file()` для удаления всего JSON-файла.  
+ в модуле **main.py** реализована головная функция, отвечающая за общую логику проекта, проверяющая работу всех представленных выше классов с большинством методов.

_Данные модули прошли проверку тестами на валидность и работоспособность!_

## Тестирование
В ходе тестирования кода был выполнен ряд проверок и отладки отдельных блоков:

**Модуль apies**  

Класс `HeadHunterAPI`:
+ Проверка наличия и работы приватного метода (обязует родительский класс);
+ Тестирование на успешное получение списка вакансий; 

**Модуль vacancy** 

Класс `Vacancy`:
+ Тестирование ограничений __slots__ и запрета на добавление новых атрибутов класса;
+ Проверка на успешное создание вакансии с зарплатой и без зарплаты;
+ Тестирование преобразования вакансии в словарь с данными;
+ Тест на сравнения зарплат вакансий;
+ Проверка диапазона зарплат на полное, частичное и отсутствие совпадений;
+ Тест с некорректным диапазоном зарплат;
+ Проверка преобразования JSON в объекты класса Vacancy;
+ Тестирование строкового представления класса.

**Модуль reports** 

Класс `JSONSaver`:
+ Проверка наличия приватного атрибута __filename в JSONSaver;
+ Тест добавления и получения вакансии в JSON-файле;
+ Проверка защиты от дубликатов в JSON-файле;
+ Тестирование удаления вакансии из JSON-файла;
+ Проверка очистки JSON-файла.

Класс `CSVSaver`:
+ Проверка инициализации CSVSaver;
+ Тестирование на добавление вакансии в CSV-файл;
+ Проверка на добавление дубликата вакансии в CSV-файл;
+ Тест получения вакансий из CSV-файла;
+ Тестирование удаления вакансии из CSV-файла;
+ Проверка очистки CSV-файла.


### Общие аспекты тестирования
**Фикстуры.** 
Все фикстуры для тестов модулей проекта перенесены в единый модуль **conftest.py**

**Mock и patch**
Тестирование функции с обращением к API, а также чтение файлов и отображение данных произведено с помощью декоратора patch и метода MagicMock().

**Покрытие тестами.** 
Все ветви кода и исключения, которые могут быть сгенерированы функциями, тестируются.
Покрытие тестами составляет 89% кода
