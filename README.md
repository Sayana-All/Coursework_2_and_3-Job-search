# Coursework2-Job-search
Курсовая по 4 модулю. Поиск вакансий с использованием API-сервиса hh.ru

## Описание
В этом проекте вам предстоит написать интеграцию с внешним сервисом.
Напишите программу, которая будет получать информацию о вакансиях с платформы hh.ru в России, сохранять ее в файл и позволять удобно работать с ней: добавлять, фильтровать, удалять.

## Информация для пользователей
### Данные
Файл с транзакциями пользователя хранится в папке data в формате Excel: [operations.xlsx](data/operations.xlsx)

```Описание данных
- Дата операции — дата, когда произошла транзакция.
- Дата платежа — дата, когда был произведен платеж.
- Номер карты — последние 4 цифры номера карты.
- Статус — статус операции (например, OK, FAILED).
- Сумма операции — сумма транзакции в оригинальной валюте.
- Валюта операции — валюта, в которой была произведена транзакция.
- Сумма платежа — сумма транзакции в валюте счета.
- Валюта платежа — валюта счета.
- Кешбэк — размер полученного кешбэка.
- Категория — категория транзакции.
- MCC — код категории транзакции (соответствует международной классификации).
- Описание — описание транзакции.
- Бонусы (включая кешбэк) — количество полученных бонусов (включая кешбэк).
- Округление на «Инвесткопилку» — сумма, которая была округлена и переведена на «Инвесткопилку».
- Сумма операции с округлением — сумма транзакции, округленная до ближайшего целого числа.
```

### В настоящее время реализовано несколько полезных функций для виджета:
+ в модуле **utils.py** реализованы вспомогательные функции для проекта и страницы _Главная_:
  * функция `get_transactions_from_excel()` реализует чтение Excel-файла (по умолчанию из `/data/operations.xlsx`) и получение списка транзакций в формате DataFrame;
  * функция `converting_data_to_json()` преобразовывает входящие данные в JSON-строку;
  * функция `user_greeting()` выводит приветствие пользователю в зависимости от времени суток;
  * функция `filter_operations()` фильтрует список банковских операций с начала месяца до входящей даты;
  * функция `get_list_cards()` возвращает список карт с номером, общей суммой и кэшбэком;
  * функция `read_user_settings` получает список пользовательских настроек для курса валют и котировок акций (по умолчанию из файла _user_settings.json_);
  * функция `currency_rate()` реализует получение курса валют к рублю через сервис Exchange Rates Data API;
  * функция `stock_prices()` реализует получение цен акций через сервис Finnhub;
  * функция `get_top_transactions()` выводит топ-5 из списка банковских операций.
+ в модуле **views.py** представлена функция _Главной_ страницы, объединяющая вспомогательные функции из модуля **utils.py**.
Возвращает JSON-ответ по заданной дате и времени.
+ в модуле **services.py** представлены услуги дополнительного сервиса для расширения основного функционала:
  * функция `calculate_increased_cashback()` проводит анализ категорий для подсчета повышенного кэшбэка;
  * функция `simple_search()` реализует простой поиск в списке транзакций по словам в категории или описании;
  * функция `search_phone_numbers()` выполняет поиск телефонных номеров в описании операций.
+ в модуле **reports.py** представлен функционал для составления отчетов и сохранения данных в файл:
  * функция-декоратор `save_file()` реализует сохранение данных отчетности в файл в формате excel(по умолчанию) или csv;
  * функция `spending_by_category()` создает отчет по тратам за последние 3 месяца с указанной даты по заданной категории.
+ в модуле **main.py** реализована головная функция, отвечающая за общую логику проекта, проверяющая работу всех представленных выше функций.

_Данные модули прошли проверку тестами на валидность и работоспособность!_
