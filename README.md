# Coursework_2_and_3-Job-search
Курсовая по 4 модулю. Поиск вакансий с использованием API-сервиса hh.ru
Курсовая по 5 модулю. Поиска вакансий с подключением БД

## Описание
В этом проекте вам предстоит написать интеграцию с внешним сервисом.
Напишите программу, которая будет получать информацию о вакансиях с платформы hh.ru в России, сохранять ее в файл и позволять удобно работать с ней: добавлять, фильтровать, удалять.
В рамках проекта вам необходимо получить данные о компаниях и вакансиях с сайта hh.ru, спроектировать таблицы в БД PostgreSQL и загрузить полученные данные в созданные таблицы.

## Информация для пользователей
### Настройка базы данных

1. Создайте файл конфигурации БД:
   ```bash
   cp config/database.ini.example config/database.ini
   ```

2. Отредактируйте `config/database.ini` и укажите свои параметры подключения:
   ```ini
   [postgresql]
   host=ваш_хост (например localhost)
   user=ваш_пользователь (например postgres)
   password=ваш_пароль
   database=имя_базы_данных (по умолчанию vacancies)
   port=5432
   ```

3. Убедитесь, что файл `database.ini` добавлен в `.gitignore`

4. Запустите код в **main.py**

### Документация для сбора вакансий с hh.ru
Ссылка на API: https://github.com/hhru/api/.


### В настоящее время реализован следующий функционал:
##### Coursework_2_Job_search
+ В модуле **apies.py** представлены классы для взаимодействия с API-сервисами:
    * создан абстрактный класс `AbstractAPI` с абстрактным методом `get_vacancies()` и `check_private_method()`, который проверяет наличие приватного метода в дочерних классах;
    * представлен дочерний класс `HeadHunterAPI` для работы с API-сервисом сайта _hh.ru_.
    Содержит приватный метод __connect_to_api переопределенный метод `get_vacancies()`, возвращающий JSON-ответ со списком вакансий.
+ В модуле **vacancy.py** представлен класс `Vacancy` для работы с вакансиями, полученными по запросу пользователя:
    * в классе определены атрибуты, такие как название вакансии (`title`), ссылка на вакансию (`url`), зарплата (`salary`) и краткое описание или требования (`description`);
    * реализованы методы валидации данных для приватных атрибутов и вывод диапазона зарплат вакансий;
    * представлены методы расчета средней зарплаты для сравнения оных у разных вакансий;
    * реализован класс-метод `cast_to_object_list()` для преобразования данных API в список объектов Vacancy;
    * есть строковое представление вакансии для показа пользователю.
+ В модуле **reports.py** представлены классы для сохранения информации о вакансиях и работы с файлами:
    * определён абстрактный класс `AbstractSaver` для работы с входящими данными. 
    Содержит абстрактные методы `add_vacancy()`, `get_vacancies()` и `delete_vacancy()` для взаимодействия с экземплярами вакансий.
    * создан дочерний класс `JSONSaver` для хранения информации в JSON-файле. В нём переопределены методы абстрактного класса, а также добавлены:
        - приватный атрибут названия файла (`filename`), который по умолчанию сохраняет данные в директорию `/data/vacancies.json`;
        - метод `load_data()` для загрузки данных из файла,
        - метод `save_data()` для сохранения данных в файл,
        - метод `clear_file()` для удаления всего JSON-файла.
    * создан дочерний класс `CSVSaver` для хранения информации в CSV-файле. В нём переопределены методы абстрактного класса, а также добавлены:
        - приватный атрибут названия файла (`filename`), который по умолчанию сохраняет данные в директорию `/data/vacancies.csv`;
        - метод `load_data()` для загрузки данных из файла,
        - метод `save_data()` для сохранения данных в файл,
        - метод `clear_file()` для удаления всего JSON-файла.  
+ в модуле **main.py** реализована головная функция, отвечающая за общую логику проекта, проверяющая работу всех представленных выше классов с большинством методов.

##### Coursework_3_Job_search_with_PostgreSQL
+ В модуле **apies.py** произведены следующие изменения:
    * абстрактный класс `AbstractAPI` теперь имеет 2 абстрактных метода: `get_employers()` и `get_employer_vacancies()`;
    * дочерний класс `HeadHunterAPI` объявлен с приватным атрибутом `__base_url` с ссылкой на URL-адрес с API-сервис и `employers`, содержащий список из 10 компаний-работодателей;
    * класс `HeadHunterAPI` содержит публичный метод `get_employers()` для получения списка компаний-работодателей и публичный метод `get_employer_vacancies()` для получения списка вакансий по ID работодателя.
+ Создана новая директория **config**, которая содержит:
    * модуль **config.py** для парсера базы данных PostgreSQL;
    * шаблон файла _database.ini.example_ с конфигурационными данными для настройки БД (инструкция описана в разделе "Информация для пользователей").
+ В новом модуле **db_manager.py** представлен класс `DBManager` для работы с данными в БД:
    * содержит атрибут с именем базы данных (`db_name`) и спец.атрибут для подключения к БД (`conn`);
    * использует маг.методы `__enter__()` с контекстным менеджером для подключения к БД и `__exit__()` для завершения соединения с БД;
    * реализован метод `create_database()` для создания БД или подключения к уже существующей;
    * реализован метод `create_tables()` для создания таблиц в БД;
    * представлен метод `insert_employer()` для добавления компании-работодателя в таблицы БД;
    * представлен метод `insert_vacancy()` для добавления вакансии в таблицы БД;
    * есть метод `get_companies_and_vacancies_count()`, который получает список всех компаний и количество вакансий у каждой компании;
    * создан метод `get_all_vacancies()` для получения списка всех вакансий с указанием названия компании, названия вакансии, зарплаты, ссылки на вакансию и её описанием;
    * реализован метод `get_avg_salary()` для получения средней зарплаты по вакансиям;
    * представлен метод `get_vacancies_with_higher_salary()` для получения списка всех вакансий, у которых зарплата выше средней по всем вакансиям;
    * есть метод `get_vacancies_with_keyword()` для получения списка всех вакансий, в названии которых содержатся переданные в метод слова, например "python";
    * есть метод `get_all_vacancies_as_objects()`, который получает все вакансии из БД и возвращает список объектов `Vacancy`;
    * реализован метод `get_vacancies_with_keyword_as_objects()` для поиска вакансий по ключевому слову с возвратом объектов `Vacancy`.
+ В модуле **vacancy.py** обновлен класс `Vacancy` со следующими улучшениями:
    * создан класс-метод `from_db_row()` для создания объекта Vacancy из строки БД;
+ В модуле **reports.py** произведены следующие обновления:
    * в классе `JSONSaver` добавлен метод `import_to_database()` для импорта вакансий из JSON-файла в базу данных;
    * в классе `CSVSaver` добавлен метод `import_to_database()` для импорта вакансий из CSV-файла в базу данных.
+ В модуле **main.py** реализован расширенный пользовательский интерфейс `user_interaction()` со следующими возможностями:
    * Загрузить вакансии выбранных компаний (`load_vacancies`);
    * Показать список компаний и количество вакансий у каждой (`show_companies`);
    * Показать все вакансии в БД (`show_all_vacancies`);
    * Показать среднюю зарплату по вакансиям (`show_avg_salary`);
    * Показать вакансии с зарплатой выше средней (`show_high_salary_vacancies`);
    * Поиск вакансий по ключевому слову (`search_vacancies`);
    * Экспорт вакансий в JSON (`export_vacancies_to_json`);
    * Экспорт вакансий в CSV (`export_vacancies_to_csv`).

_Данные модули прошли проверку тестами на валидность и работоспособность!_

## Тестирование
В ходе тестирования кода был выполнен ряд проверок и отладки отдельных блоков:

**Модуль apies**  

Класс `HeadHunterAPI`:
+ Проверка наличия и работы приватного метода (обязует родительский класс);
+ Тестирование на успешное получение списка вакансий; 

**Модуль vacancy** 

Класс `Vacancy`:
+ Тестирование ограничений __slots__ и запрета на добавление новых атрибутов класса;
+ Проверка на успешное создание вакансии с зарплатой и без зарплаты;
+ Тестирование преобразования вакансии в словарь с данными;
+ Тест на сравнения зарплат вакансий;
+ Проверка диапазона зарплат на полное, частичное и отсутствие совпадений;
+ Тест с некорректным диапазоном зарплат;
+ Проверка преобразования JSON в объекты класса Vacancy;
+ Тестирование строкового представления класса.

**Модуль reports** 

Класс `JSONSaver`:
+ Проверка наличия приватного атрибута __filename в JSONSaver;
+ Тест добавления и получения вакансии в JSON-файле;
+ Проверка защиты от дубликатов в JSON-файле;
+ Тестирование удаления вакансии из JSON-файла;
+ Проверка очистки JSON-файла.

Класс `CSVSaver`:
+ Проверка инициализации CSVSaver;
+ Тестирование на добавление вакансии в CSV-файл;
+ Проверка на добавление дубликата вакансии в CSV-файл;
+ Тест получения вакансий из CSV-файла;
+ Тестирование удаления вакансии из CSV-файла;
+ Проверка очистки CSV-файла.


### Общие аспекты тестирования
**Фикстуры.** 
Все фикстуры для тестов модулей проекта перенесены в единый модуль **conftest.py**

**Mock и patch**
Тестирование функции с обращением к API, а также чтение файлов и отображение данных произведено с помощью декоратора patch и метода MagicMock().
